- a태그 : 앵커(anchor)의 약자. 연결해주는 태그. 브라우저 주소를 대신 적어주는 기능.

- a태그에 이미지, 텍스트, 컴퓨터에 저장된 파일도 넣을 수 있음. 페이지 연결, 페이지안에서 이동도 가능.

- URL 스킴(scheme)
   - 어플마다 제공하는 주소 체계가 있음.
   - tel ; 전화 어플과 연결
   - mailto ; 메일 어플과 연결
   - tmap ; 티맵 어플과 연결 
  
**주소 체계**

1. 절대 주소
  - 프로토콜로 시작
     - http:// , https://
  - / 로 시작
  - ip:port/ 이후의 주소를 모두 변경

2. 상대 주소
   - 절대 주소가 아닌 모든 경우
   - 현재 주소의 맨 마지막 / 이후의 글씨를 모두 변경.
   - 부모 폴더 ; .. (점 두개)
   - 현재 폴더 ; . (점 한개)

- ul (Unordered List) ; 순서가 없는 목록이며, 자식 요소인 li에 컨텐츠를 넣는다.
  
- ol (Ordered List) ; 순서가 있는 목록이며, 자식 요소인 li에 컨텐츠를 넣는다.

- dl ; 용어 설명에 사용. dt : 주제 , dd : 내용 또는 설명.

- 테이블
   - 기본 양식
  <img width="623" height="154" alt="image" src="https://github.com/user-attachments/assets/f4ebc703-c789-49f2-ae5c-f9aef978c92b" />

- table : 테이블을 감싸는 태그

- tr : table row의 약자로 한 줄을 뜻함.

- td : table data의 약자로 한 칸을 뜻하며, 표시할 학몽을 적는 곳, 수직 방향 가운데 정렬을 지원.

- th : table header의 약자로 한 칸을 뜻하며, 주로 제목을 적을 때 사용한다. 수직, 수평 정렬 + 두꺼운 글씨

   - 추천 양식
  <img width="620" height="449" alt="image" src="https://github.com/user-attachments/assets/12152cf6-4c95-4a79-8688-399ffcfc97df" />

- thead : 제목 영역 / tbody : 내용 영역 (억지로 사용할 필요는 없지만, 가능하면 사용하자)

- colspan
  - 하나의 칸을 몇 칸으로 설정한건지 지정
  - 또는 칸을 합칠때 사용할 수도 있음.

- rowspan
   - 하나의 줄을 몇 줄을 설정한건지 지정
   - 또는 줄을 합칠때 사용할 수도 있음.
