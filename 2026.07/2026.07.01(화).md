## Git 

### 형상관리 (버전관리)

- 하는 일

  - 파일 공유
 
  - 수정 이력 관리
 
- 용어 설명

  - 리포지토리

    - 저장소, 저장 공간, 원격 폴더 같은 개념.

    - 줄여서 repo라고도 부름.

  - 스테이징
 
    - 신규 또는 변경된 파일을 commit 대상으로 지정.
   
  - conflict
 
    - 충돌 상황
   
    - 해결될 때까지 push 불가.
   
  - conflict 해결법 (가장 쉽고 빠른 방법)
 
    1.  일단 내 소스를 복사해놓고 지우기.
   
    2.  다른 사람 코드만 살린 상태에서 push (버전업이 됨)
   
    3.  내 코드를 다시 살려서 push

- 명령어

  - git init

    - 저장소 생성.
   
  - git add . 
 
    - 스테이징 명령어

  - git commit -m "메시지"
 
    - staging 된 파일을 업로드 대상으로 확정.
   
    - 메시지를 꼭 적는 버릇을 가지자.
   
  - git push main master
 
    - commit 된 파일을 원격 저장소(repo)로 보낸다.
   
  - git fetch
 
    - 원격 저장소의 정보로 업데이트
   
  - git pull
 
    - 변경된 파일을 다운로드.
   
  - git clone
 
    - 원격 저장소의 모든 내용을 내 폴더로 다운로드.
   
    - 내 폴더는 비어있어야 함.

  - branch
 
    -  push가 따로 관리되는 복사본.
   
    -  branch를 나누는 시점에서 폴더를 복사한다고 생각하면 편함.
   
  - merge

    - branch 합치기 (병합, 합병)
   
  - fork
 
    - branch와 같지만 push가 승인 후에 반영되는 branch
   
  - pull request
 
    - fork에 push한 내용을 승인 해 달라는 요청.

### CSS background-color 속성 

이 속성은 요소의 배경 색상을 지정한다. 색상값 또는 transparent 키워드를 지정할 수 있다.

``` css
    .bg-col-white {
      background-color: rgb(255,255,255); /* 이렇게 지정하면 배경 색상이 흰색이 된다. */
    }
```
``` css
    .bg-col-red {
      background-color: red; /* 이렇게 지정하면 배경 색상이 빨간색이 된다. */
    }
```

### CSS background 축약형

background-color, background-image, background-repeat, background-position을 한번에 적을 수 있다. 

- 순서

background : color || image || attachment || position

## CSS Font & Text 

폰트 및 텍스트 관련 속성은 폰트의 크기, 폰트의 지정, 폰트의 스타일, 텍스트 정렬 등을 정의.

### font-size 속성 : 텍스트의 크기를 정의 

``` html
  <html>
  <head>
    <style>
      .font-size-40 {font-size: 40px;}
      .font-size-2x {font-size: 2.0em;}
      .font-size-150ps {font-size: 150%;}
      .font-size-large {font-size: large;}
    </style>
  </head>
```

### font-family 속성 : 폰트를 지정하는데, 컴퓨터에 해당 폰트가 설치되어 있지 않으면 적용되지 않는다.

폰트는 여러 개를 동시에 지정이 가능하며, 첫번째 지정한 폰트가 클라이언트 컴퓨터에 설치되어 있지 않은 경우, 다음에 지정된 폰트를 적용한다. 

두번째로 지정하는 폰트는 대부분의 OS에 기본적으로 설치되어 있는 generic-family 폰트(Serif, Sans-serif, Monospace)를 지정하는 것이 일반적. 

폰트명은 따옴표로 감싸주며 폰트명이 한 단어인 경우는 따옴표로 감싸주지 않아도 된다.

``` css
.font {
    font-family : '모나12', 'Mona12', 'Mona', 'monospace';
}
```

### font-style / font-weight 속성 : font-size는 이텔릭체의 지정, font-weight는 폰트 굵기 지정에 사용된다.
``` css
.style {
    font-style : italic; /* 이렇게 하면 글씨가 기울어져서 보인다 */
}
```
```css
.fw-100 {font-weight : 100;}
.fw-300 {font-weight : 300;}
.fw-500 {font-weight : 500;} /* 100 ~ 1000까지 100단위로 지정할 수 있으며. 기본값은 500, 600 */
.fw-700 {font-weight : 700;}
.fw-900 {font-weight : 900;}
.fw-1000 {font-weight : 1000;}
```

### line-height 속성 : 텍스트의 높이를 지정하며, 수직 정렬에도 응용되어 사용.

``` css
.lh {
    line-height: 300%;
}
```

### letter-spacing 속성 : 글자 사이의 간격을 지정한다.

``` css
.spacing {
    /* 기본값 : letter-spacing: 1px;  */
    letter-spacing: -1px;
}
```

### text-align 속성 : 텍스트의 수평 정렬을 정의한다. 

``` css
.right {
    text-align: right;
}
```

### text-decoration 속성 

이 속성을 이용해서 a태그의 밑줄을 제거할 수 있으며. 

텍스트에 underline, underline, line-through를 추가할 수도 있다. 

``` css
.lt {
    text-decoration: line-through;
}

.ul {
    text-decoration: underline;
}

.none {
    text-decoration: none;
}
```

### white-space : nowarp 

- 꽉 차도 헌쥴로 표시한다. (즉, 자동 줄바꿈을 하지 않는다)

### text-overflow : ellipsis 

- 말 줄임표.

- 넘쳐서 가려지는 글이 있을떄. "..."으로 표시

### word-wrap : break-word

- 한 단어가 길어도 튀어나가지 않게 다음 줄로 줄 바꿈 해줌

- 원래 자동 줄 바꿈은 유지된다.

``` css
.word-wrap {
    word-wrap: break-word;
}
```

### word-break : break-all

- 원래 자동 줄 바꿈을 무시하고 넘치는 것들을 다음 줄로 줄 바꿈 해줌 

``` css
.word-break {
    word-break:break-all;
}
```
