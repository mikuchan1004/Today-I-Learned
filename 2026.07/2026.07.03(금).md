## CSS float

### width 속성 

이 속성의 기본값은 100%이므로, width 속성값을 지정하지 않은 블록 요소는 부모 요소의 가로폭을 가득 채운다.

width 속성을 선언하지 않은 블록 레벨 요소에 float 속성이 선언되면 width가 inline 요소와 같이 content에 맞게 

최소화되고 다음 요소 위에 떠있게 된다.

### float 속성 관련 문제 해결 방법 (이 부분은 구글 제미나이의 도움을 받아 작성하였습니다)

1. float를 쓰면 발생하는 부작용 : 자식이 공중에 떠서 부모가 찌그러지거나, 다음 박스가 밑으로 기어들어간다.

2. 부모가 찌그러질 때 해결책 : 부모 요소의 CSS에 overflow:hidden을 적거나, clearfix:after 를 사용한다.
``` css
.clearfix:after {
    content: "";
    display: block;
    clear: both;
}
```
3. 다음 박스가 파고들 때 해결책 : 파고드는 그 자식 요소의 CSS에 clear:both를 적어 아래로 밀어낸다.

### CSS shadow 

텍스트나 요소에 그림자 효과를 부여하기 위한 속성

1. text-shadow

텍스트에 그림자 효과를 부여하는 속성이다.

``` css
.red {
  text-shadow: 10px 3px red;
}
```

``` css
.blur {
    text-shadow: 1px 1px 2px red;
    color:white;
    font-size: 2em;
}
```

2. box-shadow

요소에 그림자 효과를 부여하는 속성이다.

``` css
.box {
    border: 1px solid red;
    width: 100px;
    box-shadow: 4px 4px 6px grey;
}
```

### CSS transition (이 부분은 제미나이의 도움을 받아 작성하였습니다)

1. 목적 : CSS 속성 값이 바뀔 때 순간 이동하지 않고, 지정한 시간 동안 부드러운 애니메이션으로 변신시키는 효과.

2. 트랜지션의 4가지 필수 속성

   - transition-property : 변신시킬 대상 (all, width, background-color)
  
   - transition-duration : 변신의 길이/시간 (0.5s, 2s) **[생략 시 작동 안함]**
  
   - transition-timing-function : 변신의 속도 리듬 (ease - 자연스럽게, linear - 자연스럽게)
  
   - transition-delay : 변신 전 대기 시간(1s - 1초 후 시작)

3. 한 줄 작성법

**주의 : 마우스를 뗄 때도 부드럽게 작동하게 하려면, 트랜지션 코드는 :hover가 아니라
원래 요소(기본 상태)의 CSS에 적어주어야 한다.**

``` css
.box {
    /* 순서 : 대상 변신시간 리듬 대기시간 */      
    transition: all 0.5s ease 0s; 
}
```

### CSS transform (이 부분은 제미나이의 도움을 받아 작성하였습니다)

1. 목적 : HTML 요소의 본래 물리적 위치는 유지한 채, 시각적인 형태(위치, 크기, 회전, 비틀기)만 이웃 상자 방해 없이

독립적으로 변형시키는 효과 

2. 4가지 2D 변형 함수

    - translate(x, y) : 현재 위치 기준 가로/세로 이동 (예: translate(10px, 20px))
  
    - scale(x, y) : 원래 크기(1) 기준 확대 및 축소 (예: scale(1.5) - 1.5배 확대)
  
    - rotate(각도deg) : 중심점을 기준으로 회전 (예 : rotate(45deg) - 시계방향으로 45도 회전)
  
    - skew(x, y) : 상자를 평행사변형 형태로 비틀기 (예 : skewX(15deg))
  
3. transform-origin (중심점)

   - 회전이나 변형이 일어날 때 기준이 되는 '고정 핀' 위치를 결정.
  
   - 기본값은 정중앙(50%, 50%)이며, left top 등 원하는 모서리로 변경 가능.

4. 장점 및 특징 : margin이나 top/left 이동 방식보다 웹 브라우저가 화면을 그리는 속도(성능)가 훨씬 빠르고 부드러워

애니메이션 구현 시 필수적으로 사용됨

### CSS 반응형 웹 디자인 (이 부분은 제미나이의 도움을 받아서 작성하였습니다)

1. 목적 : 하나의 HTML 코드로 PC, 태블릿, 모바일 등 다양한 기기의 화면 크기에 맞춰 레이아웃이 유연하게 변형되도록 하는 디자인 기법.

2. 반응형 웹의 3대 필수 기술

   - 뷰포트(Viewpprt) 설정 : 모바일 기기에서 화면이 축소되어 개미만 하게 보이는 현상을 막고, 실제 기기 스크린 크기에 화면을 맞추도록 설정하는 HTML 메타 태그.
  
   - 미디어 쿼리(@media) : 특정 화면 크기(Breakpoint)를 기준으로 CSS 코드를 분기하여 스마트폰/PC마다 다른 스타일이 적용되도록 하는 CSS 조건문 가능.
  
   - 유동형 레이아웃(Fluid) : 고정된 px 단위를 지양하고, %, vw, vh 등 상대 단위를 사용하여 화면 크기에 맞춰 컨텐트가 신축성 있게 늘어나고 줄어들게 구현.
  
3. 반응형 이미지 적용 CSS 코드
``` css
img {
        max-width: 100%; /* 이미지 부모 박스 크기 초과 방지 */
        height : auto; /* 원본 가로세로 비율 유지 */
]
```

4. 장점 : 기기별로 사이트를 따로 만들 필요가 없어서 유지보수가 매우 편리.

### CSS flexbox (이 부분은 제미나이의 도움을 받아서 작성하였습니다)

1. 플렉스박스 레이아웃 체계 : 전체 정렬을 지휘하는 부모(Container)와 개별 속성을 가지는 자식(Item)의 규칙으로 나눠 설계됨.

2. 부모(Container)의 핵심 속성

   - display: flex; : flexbox 활성화
  
   - flex-direction : 가로 (row) 또는 세로 (column) 흐름 결정.
  
   - flex-wrap : 자식 요소의 줄바꿈 여부 (wrap / nowrap).
  
   - justify-content : 메인 축 기준 정렬 (center, space-between 등).
  
   - align-items : 수직 축 기준 정렬 (center, stretch 등)
  
3. 자식(Item)의 핵심 속성 (유연성 조절)

   - flex-grow : 남는 공간을 나눠 가질 크기 확장 비율 (기본값: 0 - 안 커짐)
  
   - flex-shrink : 공간 부족 시 줄어들 크기 축소 비율 (기본값: 1 - 같이 줄어듦)
  
   - align-self : 부모의 수직 정렬 규칙을 무시하고 해당 자식만 독립적으로 수직 정렬.

4. 자식(Item) 단축 표기법
``` css
.item {
    /* 순서 : grow (확장) shrink (축소) basis (기본크기) */
    flex 1 1 auto;
}
```
